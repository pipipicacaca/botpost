# 🤖 PostBuilder Bot

Конструктор постов в **Telegram Rich Markdown** (Bot API 10.1, обновление 11.06.2026).
Человек собирает пост из блоков по кнопкам и пишет текст руками — бот рендерит в
Rich Markdown и отправляет как Rich Message, который Telegram рисует нативно:
заголовки, таблицы, LaTeX, чеклисты, code, цитаты, спойлеры. Медиа уходят отдельно.

## Типы блоков

Текст и структура: заголовок (H1–H6), абзац, список, нумерованный список,
чеклист, цитата, разделитель.
Продвинутое: таблица (GFM), формула (LaTeX), код (с языком), pull-quote,
collapsible-секция (спойлер).
Медиа: фото (загрузка/поиск), видео, аудио, коллаж (несколько фото), карта.

## Ввод блоков (подсказки бот даёт сам)

- **Заголовок**: «2: Текст» → H2. Без цифры — H2 по умолчанию.
- **Чеклист**: пункты по строкам; «+ » в начале строки = отмеченный.
- **Код**: первая строка «lang: python» задаёт язык подсветки.
- **Таблица**: ряды по строкам, ячейки через «;». Первый ряд — шапка.
- **Формула**: чистый LaTeX без знаков $ (бот обернёт в `$$`).
- **Спойлер**: первая строка — заголовок, дальше — содержимое.
- **Карта**: геопозиция через скрепку или координаты «55.75, 37.61».

## Сценарий

1. `/start`
2. ➕ Добавить блок → выбрать тип → ввести текст/медиа
3. ✏️ Редактировать → выбрать блок → изменить / ⬆️⬇️ / удалить
4. 👁 Предпросмотр — структура
5. 📤 Собрать пост — бот шлёт медиа + Rich Message-текст

## Структура

```
postbot/
├── bot.py          # хендлеры, FSM, точка входа
├── database.py     # SQLite (WAL): посты и блоки (+ extra JSON)
├── renderer.py     # блоки → Rich Markdown (GFM)
├── sender.py       # отправка Rich Message + fallback  ← СВЕРИТЬ С ДОКОЙ
├── keyboards.py    # инлайн-клавиатуры
├── photo_search.py # поиск фото
├── requirements.txt
└── Procfile
```

## Отправка Rich Message (схема подтверждена)

Готовый пост уходит методом `sendRichMessage` (Bot API 10.1). Тело запроса:

```json
{"chat_id": <id>, "rich_message": {"markdown": "<GFM-текст>"}}
```

`InputRichMessage` требует ровно ОДНО из полей — `markdown` или `html`.
Мы кладём GFM в `markdown`. Логика — в `sender.py`. При ошибке отправки
бот делает fallback на обычное сообщение и пишет причину в лог.

## Медиа в посте (хостинг картинок)

Rich message требует постоянный публичный URL картинки. Бот заливает
загруженные фото/видео на публичный хостинг и встраивает в единый пост:

- фото → `![](url "подпись")`
- коллаж → `<tg-collage> ![](u1) ![](u2) </tg-collage>`
- карта → `<tg-map lat=".." long=".." zoom="14"/>`

Логика заливки — `telegraph_upload.py`:
- если задан `IMGBB_KEY` → грузит на imgbb (стабильнее, для фото);
- иначе → Catbox.moe (без ключа, фото и видео).

`IMGBB_KEY` (опционально) — бесплатный ключ с https://api.imgbb.com/.
Без ключа всё работает через Catbox.

⚠️ Аудио пока уходит отдельным сообщением. Если хостинг отдаёт прямой URL
аудио — звук тоже можно встроить (см. RichBlockAudio в доке).

```bash
pip install -r requirements.txt
export BOT_TOKEN="токен_от_@BotFather"
python bot.py
```

## Деплой на Railway

1. Залить папку в GitHub
2. Railway → New Project → Deploy from GitHub
3. Variables: `BOT_TOKEN` (и опц. `UNSPLASH_ACCESS_KEY` для поиска фото)
4. Railway поднимет worker по `Procfile`
